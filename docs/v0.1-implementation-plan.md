# Haiti City Portal — Implementation Status

This document tracks what has been built and what remains. It replaces the original milestone-based pre-build plan with a current-state audit.

---

## Current Status (as of March 2026)

The v0.1 foundation is complete. The app is a functioning multilingual civic portal with real content, multi-tenant routing, an admin panel, a payment infrastructure, and a service request system.

## What Remains (Backlog)

### High Priority

- [ ] **Offline draft mode**: `useOfflineQueue` hook saving form submissions to IndexedDB. Service worker background sync when the connection is restored. Required for low-bandwidth Haiti use.
- [ ] **MonCash live integration**: Webhook handler at `/api/payments/moncash/webhook` to auto-confirm payments instead of manual reconciliation.
- [ ] **More news articles**: The current 3 sample articles need to be replaced with real municipal content in all 4 languages.
- [ ] **PWA manifest + service worker**: `public/manifest.json`, asset caching strategy (cache-first for static, network-first for API).

### Medium Priority

- [ ] **Stripe payment gateway**: USD/EUR checkout for diaspora users.
- [ ] **Portuguese (pt-BR)**: Fifth language. Add `pt` to routing, create `messages/pt.json`, and create `.pt.mdx` content files.
- [ ] **E2E test suite**: Playwright tests for: login flow, service request submission, payment flow, language switching.
- [ ] **CI/CD**: GitHub Actions workflow for lint + type-check + build on every PR.
- [ ] **User registration**: Allow residents to create an account to track their requests.

### Low Priority / Future

- [ ] **Blockchain audit trail**: SHA-256 hash of finalized records anchored to Polygon/Ethereum. Badge UI for verified records.
- [ ] **WhatsApp bot integration**: Automated status update messages.
- [ ] **GIS property lookup**: OpenStreetMap parcel layer for the tax lookup.
- [ ] **Mobile app**: React Native shell wrapping the portal.
- [ ] **Satellite sync**: Batch sync for remote town halls via Starlink.

## Adding Content

### New news article

1. Create `src/content/news/YYYY-MM-DD-{slug}.mdx` with:
   ```yaml
   ---
   date: "Month DD, YYYY"
   dateISO: "YYYY-MM-DD"
   title: "Article title"
   description: "Short excerpt."
   ---
   Full article body.
   ```
2. Create `src/content/news/YYYY-MM-DD-{slug}.fr.mdx`, `.ht.mdx`, `.es.mdx` (all optional — falls back to English).
3. No code changes needed. The article appears on the homepage and news index automatically.

### New service page

1. Create `src/content/services/{slug}.mdx` with the service frontmatter schema.
2. Create locale variants as needed.
3. Create `src/app/[locale]/(public)/services/{slug}/page.tsx` — use `loadContent("services/{slug}", locale)` and render with `<ServiceInfoPage />`.
4. Add a card to the services directory page.

### Updating existing content

Edit the appropriate `.mdx` file. Changes are picked up on the next request (no build step required in development). For production, a redeploy is needed since content is read at request time from the filesystem.
